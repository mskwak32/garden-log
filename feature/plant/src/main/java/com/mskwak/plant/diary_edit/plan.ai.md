# 일기(diary)추가 기능

일기 추가/편집 화면.

## 진입점
- 일기 추가 : PlantDetailScreen의 일기 쓰기 버튼으로 진입
- 일기 수정 : DiaryDetailScreen에서 메뉴 > 수정으로 진입
  -  DiaryDetailScreen은 아직 만들어지지 않았음. 추후 예정

## 기능 명세

- 사진 추가
  - 권한 여부 체크
    - 권한 거부 시 동작은 plantEditScreen과 동일하게 처리
  - 사진은 5장까지 추가
    - 5장 이상 추가하려고 할 경우 "더이상 이미지를 추가할 수 없습니다." snackbar
  - SelectPhotoDialog 활용
    - 삭제버튼 없음
  - 추가된 사진의 X 아이콘 표시
    - 클릭 시 추가된 사진 삭제
  - 사진은 가로 스크롤 레이아웃
  - 사진의 썸네일의 크기 : 80dp x 80dp
- 날짜 표시
  - default: 오늘 날짜
  - 클릭 시에 날짜 변경 가능
  - 미래 날짜도 선택 가능
- 일기 입력
  - hint: 내용을 입력해주세요.
  - 하단의 30dp 정도를 남기고 화면의 나머지 높이 차지
    - TextField 세로 스크롤
  - 사진이나 일기 내용 둘 중 하나는 있어야 함.
    - 둘 다 없는 상태로 저장 버튼 클릭 시 "사진이나 내용을 입력해주세요" Snackbar 표시
- TopBar
  - 가운데에는 PlantName 출력
  - 왼쪽 백버튼 또는 X(닫기) 버튼
  - 오른쪽에 저장/수정 버튼으로 일기 저장
- 사진은 선택하면 파일로 저장하고 사진을 삭제하거나 뒤로가기/백버튼으로 화면을 닫을 경우 저장했던 파일 삭제
  - PlantEdit의 사진저장 로직 참고
- 뒤로 가기 시 사진이나 내용이 있을 경우 "작성 중인 내용이 삭제됩니다." 확인 다이얼로그 필요
